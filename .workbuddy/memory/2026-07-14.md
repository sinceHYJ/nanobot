## 架构澄清：CLI / Gateway / Channel 三者关系
- CLI (`nanobot agent`) 与 Gateway (`nanobot gateway` / `nanobot webui`) 是两种**独立进程**，底层都复用同一套 `AgentLoop` + `MessageBus` 核心。
- **Channel 不是独立入口**，而是 Gateway 内部由 `ChannelManager` 加载的消息适配器组件，依附 Gateway 运行，不能单独"运行一个 channel"。
- WebUI (`nanobot webui`) 本质 = Gateway 的 **WebSocket channel**（承载前端）+ 打开浏览器。
- 另有 `nanobot serve`（OpenAI 兼容 API，独立进程但不过 ChannelManager）和 Python SDK 两种入口。
- 关键代码位置：`nanobot/cli/commands.py`（agent/gateway/serve/webui 命令）、`nanobot/channels/manager.py`（ChannelManager + `_dispatch_outbound`）、`nanobot/bus/queue.py`（MessageBus）。

## 本地运行方式（Windows / 源码）
- 安装（uv 推荐路径，项目有 uv.lock）：`cd D:\cp\project\nanobot && uv sync`（建 .venv + 可编辑装自身），然后一律用 `uv run nanobot ...` 运行（保证走 .venv，不依赖 PATH）。可加 extra：`uv sync --extra telegram` / `--extra dev` / `--all-extras`。验证 `uv run nanobot --version`。
- 安装（pip 路径）：`python -m pip install -e .`（可编辑，改代码即时生效，适合学习）；或 `uv pip install -e .`。
- 安装（PyPI 稳定版，无关源码）：`uv tool install nanobot-ai` 然后 `nanobot ...`。
- 初始化：`nanobot onboard`（生成 `C:\Users\heyingjie\.nanobot\config.json` + `workspace/`）。
- 配模型：在 config.json 加 `providers.<name>` + `modelPresets.primary`（provider/model/maxTokens/contextWindowTokens）+ `agents.defaults.modelPreset`。可用 generic `custom`(OpenAI 兼容) 或本地 `ollama`(apiBase http://localhost:11434/v1，免 key)。
- 启动：WebUI `nanobot webui`（开 http://127.0.0.1:8765，仅绑 127.0.0.1；`--background` 后台）；终端 `nanobot agent -m "..."` / `nanobot agent`。
- 端口：8765=WebUI/WebSocket，18790=gateway 健康检查。
- 前端开发：`cd webui && bun install && bun run dev`，代理到 gateway :8765。
